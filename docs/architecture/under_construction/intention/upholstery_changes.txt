I have made crucial changes to the upholstery architecture in the backend.

now there is upholstery category linkage as a parent.

this will change the way a user searches and selects upholstery, so as how and upholstery is created.

we will first allow for the creation of an ulpholstery category, this will be done through the upholstery inventory page, using the fab button of that page, we will add one more option to the "+" options, which is the ability to open the upholstery cateogry form page.
This creation page will follow the same architecture and design as the creation of upholstery page, this page will be rendered on the slide page surface, the user can add the name, image_url ( with preview ), mark it as favorite .
the endpoint that can be used for creating an upholstery category is documented at the handoff guide.

when creating an upholstery instance the upholstery creation form will now have a field where the user can select the upholstery category.
for this we will make the current upholstery creation form to be a staged form, using the primitive stage form component.

The first staged step for the creation of the upholstery instance is the selection of upholstery category.
The second staged step is the current form of upholstery instance creation.

For the selection of upholstery category we will create a component like the internal form working section picker.
this component will fetch and handle it's own query cycle ( endpoint documented in the handoff ).
that component is made out of a header and a body. The header will have the searchbar ( we use the primitive searchbar with no filter nor sorting icon activated ), the input changes the query param "q". The body renders a list of upholstery category cards, this upholstery cateogry cards render the upholstery_category.image_url, upholstery_category.name, count of upholstery linkages under it. the upholstery image renders in a circular avatar.
the user can only select one upholstery category instance, this selection marks the card with the bg-primary ( text turns into text-card ). the selection of an upholstery category will automatically take the user to the next upholstery staged form step. the upholstery_category.image_url will auto fill the upholstery.image_url, the user can change it but as default it does that. 
submiting the creation closed the slide page.

For the update of an upholstery instance we will keep the same pattern as creation, with the staged form and the filled information, it first opens on the step two, but if the user goes to step 1 it will see the selected upholstery category, this is because the component got the upholstery_category with the get call with the upholstery category id and filled up the searchbar with the name of the upholstery category ( that mode should not trigger the searchbar query when filling up the searchbar after the fetch of the upholstery, as it will be unecessary double fetch ).
